# Jenkins+Docker+SpringCloud持续集成流程说明

大致流程说明：

1）开发人员每天把代码提交到Gitlab代码仓库

2）Jenkins从Gitlab中拉取项目源码，编译并打成jar包，然后构建成Docker镜像，将镜像上传到 Harbor私有仓库。

3）Jenkins发送SSH远程命令，让生产部署服务器到Harbor私有仓库拉取镜像到本地，然后创建容器。

4）最后，用户可以访问到容器

# 设置SSH通过密钥登录

提示

理论上只需要在生产服务器配置SSH密钥登录，最后拷贝Jenkins服务器上的公钥即可。

1）生成密钥，并通过-m PEM指定格式必须

ssh-keygen -m PEM -t rsa -b 4096

参数说明：

  • -m 参数指定密钥的格式，PEM是rsa之前使用的旧格式
  • -b 指定密钥长度。对于RSA密钥，最小要求768位，默认是2048位。

查看密钥格式


 



ubuntu@jenkins:~/.ssh$ more /home/ubuntu/.ssh/id_rsa
-----BEGIN RSA PRIVATE KEY-----
MIIJKQIBAAKCAgEAuL2pM5NMsPPR9LhvNbfjx1uVMSR8z98ID/s1qMQ5bYVbYTIg
...

可以看到密钥的首行是：

-----BEGIN RSA PRIVATE KEY-----

而我们生成密钥时如果不指定格式，默认生成的密钥首行是：

-----BEGIN OPENSSH PRIVATE KEY-----

而 Jenkins 2.289.1 版本在检验密钥时还不支持这种格式，会出现如下错误：

jenkins.plugins.publish_over.BapPublisherException: Failed to add SSH key. Message [invalid privatekey: [B@60373f7]

键入以下命令，在服务器上安装公钥：

cd ~/.ssh
cat id_rsa.pub >> authorized_keys

如此便完成了公钥的安装。为了确保连接成功，请保证以下文件权限正确：

chmod 600 authorized_keys
chmod 700 ~/.ssh

设置SSH，打开密钥登录功能

编辑 vi /etc/ssh/sshd_config 文件，进行如下设置：

PubkeyAuthentication yes

当你完成全部设置，并以密钥方式登录成功后，再禁用密码登录：

PasswordAuthentication no

最后，将私钥文件 id_rsa 下载到客户端机器上，重启 SSH 服务：

systemctl restart sshd.service

登录jenkins服务器，拷贝ssh公钥到生产服务器

ubuntu@jenkins:~$ ssh-copy-id ubuntu@10.17.10.103

# 安装 Publish Over SSH 插件

安装以下插件，可以实现远程发送Shell命令

提示

在配置一下内容时，要首先在jenkins服务器上生成私钥，然后将公钥推送到生成服务器上。

登录jenkins服务器，拷贝私钥文件id_rsa到jenkins工作目录。

# 切换root用户
ubuntu@jenkins:~$ su
# 拷贝私钥文件
root@jenkins:/home/ubuntu# cp /home/ubuntu/.ssh/id_rsa /var/lib/jenkins/.ssh
# 修改私钥文件访问权限
root@jenkins:/home/ubuntu# chown jenkins:jenkins /var/lib/jenkins/.ssh/id_rsa

系统管理 -> 系统配置 -> Publish over SSH

  • Passphrase： 如果私钥设置了密码就是私钥的密码，私钥没设置密码可以不填
  • Path to key： 私钥的位置(实际文件所在位置/var/lib/jenkins/.ssh/id_rsa)
  • Key： 私钥的内容。如果此处填入了值，则以此处的值为准，会忽略掉Path to key的配置

SSH Servers：服务器的配置

  • Name： 名称（自定义）
  • Hostname： 服务器地址
  • Username： 用户名
  • Remote Directory： 默认远程服务器的地址

# Jenkins中Pipeline没有Docker执行权限解决办法

vi /etc/init.d/jenkins

添加如下内容到脚本顶部

DOCKER_SOCKET=/var/run/docker.sock
DOCKER_GROUP=docker
JENKINS_USER=jenkins

if [ -S ${DOCKER_SOCKET} ]; then
    DOCKER_GID=$(stat -c '%g' ${DOCKER_SOCKET})
    sudo groupadd -for -g ${DOCKER_GID} ${DOCKER_GROUP}
    sudo usermod -aG ${DOCKER_GROUP} ${JENKINS_USER}
fi

# Start Jenkins service
sudo service jenkins restart

# 编写deploy.sh部署脚本

上传deploy.sh文件到/opt/jenkins_shell目录下，且文件至少有执行权限！

#!/bin/bash
  
harbor_url=$1
harbor_project_name=$2
project_name=$3
tag=$4
port=$5
harbor_username=$6
harbor_password=$7

image_name="${harbor_url}/${harbor_project_name}/${project_name}:${tag}"


containerId=`docker ps -a | grep -w ${project_name}:${tag} | awk '{print $1}'`

if [ "$containerId" != "" ]; then
    docker stop $containerId
    docker rm $containerId
    echo "Container deleted successfully"
fi

docker login -u ${harbor_username} -p ${harbor_password} ${harbor_url}

docker pull $image_name

docker run -itd -p ${port}:${port} --name ${project_name} ${image_name}

docker rmi $(docker images -q -f dangling=true)

echo "Deployment complete"

添加执行权限

chmod +x deploy.sh
