如何使用 TLS 保护对 MinIO 服务器的访问

# 如何使用 TLS 保护对 MinIO 服务器的访问

本指南解释了如何在 Linux 和 Windows 平台上使用 TLS 证书配置 MinIO Server。

# 1.安装MinIO服务器

使用MinIO 快速入门指南中的说明安装 MinIO Server 。

# 2. 在 MinIO 中使用现有的密钥和证书

本节介绍如何使用从证书颁发机构 (CA) 获得的私钥和公共证书。如果尚未获取这些文件，请跳至3. 生成自签名证书 (opens new window)或使用以下说明使用Let's Encrypt (opens new window)生成它们：使用 Certbot for MinIO 生成 Let's Encrypt 证书 (opens new window)。

将现有的私钥和公共证书复制到certs目录中。默认的 certs 目录是：

  • Linux： ${HOME}/.minio/certs
  • 视窗： %%USERPROFILE%%\.minio\certs

笔记：

  • 可以使用--certs-dir命令行选项指定自定义证书目录的位置。
  • 在certs目录中，私钥必须命名private.key，公钥必须命名public.crt。
  • 由 CA 签署的证书包含有关颁发的身份（例如名称、到期时间、公钥）和任何中间证书的信息。不包括根 CA。

# 3. 使用 MinIO 生成和使用自签名密钥和证书

本节介绍如何使用各种工具生成自签名证书：

  • 3.1 使用certgen生成证书 (opens new window)
  • 3.2 使用 OpenSSL 生成证书 (opens new window)
  • 3.3 使用OpenSSL（带IP地址）生成证书 (opens new window)
  • 3.4 使用 GnuTLS (for Windows) 生成证书 (opens new window)

说明：

  • MinIO 在 Linux 和 Windows 上仅支持 PEM 格式的密钥和证书。
  • MinIO 目前不支持 PFX 证书。

# 3.1certgen用于生成证书

下载certgen (opens new window)适用于您的特定操作系统和平台。

certgen是一个简单的Go工具来生成自签名证书，并提供带有 DNS 和 IP 条目的 SAN 证书：

./certgen -ca -host "10.10.0.3,10.10.0.4,10.10.0.5"

应显示与此类似的响应：

2018/11/21 10:16:18 wrote public.crt
2018/11/21 10:16:18 wrote private.key

# 3.2 使用 OpenSSL 生成证书

使用以下方法之一生成证书openssl：

  • 3.2.1用ECDSA生成私钥 (opens new window)
  • 3.2.2用RSA生成私钥 (opens new window)
  • 3.2.3生成自签名证书 (opens new window)

# 3.2.1 使用 ECDSA 生成私钥。

使用以下命令生成带有 ECDSA 的私钥：

openssl ecparam -genkey -name prime256v1 | openssl ec -out private.key

应显示与此类似的响应：

read EC key
writing EC key

或者，使用以下命令生成受密码保护的私有 ECDSA 密钥：

openssl ecparam -genkey -name prime256v1 | openssl ec -aes256 -out private.key -passout pass:PASSWORD

注意

目前不支持 NIST 曲线 P-384 和 P-521。

# 3.2.2 使用 RSA 生成私钥。

使用以下命令生成带有 RSA 的私钥：

openssl genrsa -out private.key 2048

应显示与此类似的响应：

Generating RSA private key, 2048 bit long modulus
............................................+++
...........+++
e is 65537 (0x10001)

或者，使用以下命令生成受密码保护的私有 RSA 密钥：

openssl genrsa -aes256 -passout pass:PASSWORD -out private.key 2048

注意

使用受密码保护的私钥时，必须MINIO_CERT_PASSWD使用以下命令通过环境变量提供密码：

export MINIO_CERT_PASSWD=<PASSWORD>

私有加密密钥的默认 OpenSSL 格式是 PKCS-8，但 MinIO 仅支持 PKCS-1。可以使用以下命令将已使用 PKCS-8 格式化的 RSA 密钥转换为 PKCS-1：

openssl rsa -in private-pkcs8-key.key -aes256 -passout pass:PASSWORD -out private.key

# 3.2.3 生成自签名证书。

创建一个以openssl.conf以下内容命名的文件。设置IP.1和/或DNS.1指向正确的 IP/DNS 地址：

[req]
distinguished_name = req_distinguished_name
x509_extensions = v3_req
prompt = no

[req_distinguished_name]
C = US
ST = VA
L = Somewhere
O = MyOrg
OU = MyOU
CN = MyServerName

[v3_req]
subjectAltName = @alt_names

[alt_names]
IP.1 = 127.0.0.1
DNS.1 = localhost

openssl通过指定配置文件运行并在出现提示时输入密码：

openssl req -new -x509 -nodes -days 730 -key private.key -out public.crt -config openssl.conf

# 3.3 使用 GnuTLS (for Windows) 生成证书

本节介绍如何在 Windows 上使用 GnuTLS 生成证书。

# 3.3.1 安装和配置 GnuTLS。

从这里 (opens new window)下载并解压 Windows 版本的 GnuTLS 。

使用 PowerShell 将提取的 GnuTLS 二进制文件的路径添加到系统路径：

setx path "%path%;C:\Users\MyUser\Downloads\gnutls-3.4.9-w64\bin"

注意

可能需要重新启动 PowerShell 才能使此更改生效。

# 3.3.2 生成私钥：

运行以下命令生成私有.key文件：

certtool.exe --generate-privkey --outfile private.key

应显示与此类似的响应：

Generating a 3072 bit RSA private key...

# 3.3.3 生成公共证书：

创建一个cert.cnf使用以下内容调用的文件。此文件包含使用certtool.exe以下命令生成证书所需的所有信息：

# X.509 Certificate options
#
# DN options

# The organization of the subject.
organization = "Example Inc."

# The organizational unit of the subject.
#unit = "sleeping dept."

# The state of the certificate owner.
state = "Example"

# The country of the subject. Two letter code.
country = "EX"

# The common name of the certificate owner.
cn = "Sally Certowner"

# In how many days, counting from today, this certificate will expire.
expiration_days = 365

# X.509 v3 extensions

# DNS name(s) of the server
dns_name = "localhost"

# (Optional) Server IP address
ip_address = "127.0.0.1"

# Whether this certificate will be used for a TLS server
tls_www_server

运行certtool.exe并指定配置文件生成证书：

certtool.exe --generate-self-signed --load-privkey private.key --template cert.cnf --outfile public.crt

# 4. 安装来自第三方 CA 的证书

MinIO 可以连接到其他服务器，包括 MinIO 节点或其他服务器类型，例如 NAT 和 Redis。如果这些服务器使用未在已知 CA 注册的证书，请将这些证书置于以下 MinIO 配置路径之一下，以将这些证书的信任添加到 MinIO 服务器：

  • Linux： ~/.minio/certs/CAs/
  • Windods：C:\Users\<Username>\.minio\certs\CAs
