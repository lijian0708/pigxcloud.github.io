CentOS7 离线安装MongoDB

# CentOS7 离线安装MongoDB

MongoDB 提供了 linux 各个发行版本 64 位的安装包，你可以在官网下载安装包。

# 软件版本

依赖 版本
操作系统 CentOS Linux release 7.6.1810 (Core)
MongoDB 4.4.2

# 安装依赖 非必须

sudo yum install libcurl openssl

# 源码下载

前往MongoDB官网 (opens new window)下载所需的MongoDB版本，本次以当前最新版本MongoDB 4.4.2为例进行安装。

点击Download按钮直接下载或使用wget下载

wget https://fastdl.mongodb.org/linux/mongodb-linux-x86_64-rhel70-4.4.2.tgz

# 准备安装

解压

tar -zxvf mongodb-linux-x86_64-rhel70-4.4.2.tgz

将解压包拷贝到指定目录

mv mongodb-linux-x86_64-rhel70-4.4.2 /usr/local/mongodb

MongoDB 的可执行文件位于 bin 目录下，所以可以将其添加到 PATH 路径中：

echo 'export PATH=/usr/local/mongodb/bin:$PATH' >> /etc/profile

更新

source /etc/profile

# 创建数据目录

建立data文件夹，用于存放数据文件

mkdir -p /usr/local/mongodb/data/db

建立logs文件夹，用于存放日志文件

mkdir /usr/local/mongodb/logs

建立conf文件夹，用于存放配置文件

mkdir /usr/local/mongodb/conf

# 配置mongodb.conf文件

进入conf目录，执行如下命令建立配置文件

vi /usr/local/mongodb/conf/mongodb.conf

添加如下内容：

port=27017
dbpath=/usr/local/mongodb/data/db
logpath=/usr/local/mongodb/logs/mongodb.log
fork=true
auth=true
bind_ip = 0.0.0.0

mongodb.conf文件描述如下：

名称 描述
port 监听的端口
dbpath 数据存放路径
logpath 日志存放路径
fork 是否启用后台运行 1. true启用后台运行 2. false禁止后台运行
auth 是否需要验证登录权限登录（是否需要用户名密码）
bind_ip 绑定IP地址，可以写成本机的IP也可以写成0.0.0.0，写成0.0.0.0的话就会监听所有的地址

# 配置开机自启动

创建mongodb.service文件

vi /lib/systemd/system/mongodb.service

添加如下内容：








 

 





[Unit]
 
Description=mongodb
After=network.target remote-fs.target nss-lookup.target
 
[Service]
Type=forking
ExecStart=/usr/local/mongodb/bin/mongod -f /usr/local/mongodb/conf/mongodb.conf
ExecReload=/bin/kill -s HUP $MAINPID
ExecStop=/usr/local/mongodb/bin/mongod --shutdown -f /usr/local/mongodb/conf/mongodb.conf
PrivateTmp=true
 
[Install]
WantedBy=multi-user.target

提示

其中ExecStart参数与ExecStop参数中的路径要和安装的路径一致

重新加载服务配置文件

systemctl daemon-reload

添加至开机启动

systemctl enable mongodb.service

# 操作命令

启动服务

systemctl start mongodb.service

停止服务

systemctl stop mongodb.service

重启服务

systemctl restart mongodb.service

查看服务状态

systemctl status mongodb.service

# 添加防火墙例外

使用root用户执行命令 开启27017 端口

firewall-cmd --zone=public --add-port=27017/tcp --permanent

重新加载

firewall-cmd --reload

# 创建管理账户

直接使用mongo命令进行连接，默认端口是27017

mongo

创建验证用户, 提示rootpassword替换成你自己的密码

> use admin
> db.createUser({user:"root",pwd:"rootpassword",roles:[{role:"root",db:"admin"}]})

登录验证，提示rootpassword替换成你自己的密码

mongo -u root -p rootpassword --authenticationDatabase admin

# 创建普通账户

使用用户管理账户登录认证，提示rootpassword替换成你自己的密码

mongo
> use admin
> db.auth('root', 'rootpassword')
1

切换数据库（保持管理账户登录状态）

> use ivoov

提示

使用命令：use ivoov切换数据库，如果数据库不存在则自动创建。

建立普通账户（保持管理账户登录状态）

> db.createUser(
{
    "user": "ivoov",
    "pwd": "ivoov.com",
    "roles": [
        {
            "role": "readWrite",
            "db": "ivoov"
        }
    ]
})

提示

该操作创建了一个用户名为ivoov,密码为ivoov.com,所属数据库为ivoovdb 并赋予读写权限。

普通用户登录

> db.auth('ivoov', 'ivoov.com')
1
