基于 Docker 安装 Nexus

# 基于 Docker 安装 Nexus

# 系统需求

# CPU

性能主要受 I/O (磁盘和网络)而不是 CPU 的限制。 可用的 cpu 会影响较长时间运行的操作，也会影响 web 容器的线程分配算法。

  • 最小 CPUs: 4
  • 推荐 CPUs: 8+

# 内存

  • 最小 4GB
  • 官方资料 (opens new window)

# 版本说明

Sonatype Nexus Repository Manager 3的 Dockerfile，从3.18开始，这个映像基于 Red Hat Universal Base Image，而早期版本使用 CentOS。

# 使用 Docker 安装 Nexus

我们使用 Docker 来安装和运行 Nexus，使用 Docker 处理持久性存储需求通常有两种方法。

# 方法一 使用docker 数据卷方式 推荐

使用docker 数据卷方式docker-compose.yml 配置如下：













 


 

version: "3.7"
services:

  nexus:
    restart: always
    image: sonatype/nexus3
    container_name: nexus
    environment:
      NEXUS_CONTEXT: nexus
    ports:
      - "8081:8081"
    volumes:
      - nexus-data:/nexus-data

volumes:
  nexus-data:
  • nexus 默认对外暴露端口8081.
  • 使用环境变量自定义 Nexus 的上下文路径NEXUS_CONTEXT，可选 默认：/。
  • 持久目录 /nexus-data 用于配置、日志和存储。Nexus 进程需要对这个目录进行写操作，它以 UID 200的形式运行。

# 方法二 使用挂载主机目录方式

将主机目录挂载为卷。 这是不可移植的，因为它依赖于对主机具有正确权限的现有目录。 但是，在某些情况下，如果需要将该卷分配给某些特定的底层存储，那么它会很有用。

1.创建数据卷目录，并修改目录权限。

mkdir /usr/local/docker/nexus/nexus-data && chown -R 200 /usr/local/docker/nexus/nexus-data

2.使用docker 挂载主机目录方式docker-compose.yml 配置如下：













 

version: "3.7"
services:

  nexus:
    restart: always
    image: sonatype/nexus3
    container_name: nexus
    environment:
      NEXUS_CONTEXT: nexus
    ports:
      - "8081:8081"
    volumes:
      - /usr/local/docker/nexus/data:/nexus-data

# 启动服务

docker-compose up -d

在一个新的容器中启动服务可能需要一些时间(2-3分钟)。你可以通过跟踪日志来确定 Nexus 是否准备好了。

# 查看日志

docker-compose logs -f nexus

# 登录控制台验证安装

  • 浏览器访问地址：http://ivoov.com:8081/nexus，看到如下页面说明 Nexus 服务启动成功了。

# 获取管理员密码

默认用户是 admin，生成的密码可以在卷中的 admin.password 文件中找到。

cat /var/lib/docker/volumes/nexus_nexus-data/_data/admin.password

例如：23896e46-96bf-4b7f-8332-1bd45e32bea5

# 登录Nexus控制台

首次登录nexus控制台，管理用户是admin，密码输入上一步admin.password中获取的初始密码。

进入系统后，根据系统提示重新设置管理密码即可。

# 删除 Nexus

进入docker-compose.yml文件所在目录，执行如下命令：

docker-compose down --volume
  • 参数--volume可选，指定是否删除数据卷。

# Nginx 反向代理 Nexus

Nginx 反向代理 nexus 的服务, 一直卡在 Initialize...

解决方式是添加一个 header X-Forwarded-Proto:

proxy_set_header X-Forwarded-Proto "https";

nexus.conf

upstream nexus-server{
    server 127.0.0.1:8081;
}

server {
    listen 80;
    server_name xx.xx;
    location / {
        return 301 https://xx.xx;
    }

    location ~ /.well-known {
        root /tmp;
    }
}

server {
    listen 443 ssl;
    server_name xx.xx;

    ssl_certificate cert/xx.pem;
    ssl_certificate_key cert/xx.key;

    location / {
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-Proto "https";
        proxy_pass http://nexus-server;
    }
}
